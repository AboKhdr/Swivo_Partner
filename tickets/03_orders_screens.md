# 03 — Orders Screens

## الشاشات المشمولة
1. `OrdersListScreen` — قائمة كل الطلبات (نشطة / سابقة)
2. `OrderDetailsScreen` — تفاصيل طلب واحد + تقدم الحالة + الإجراءات
3. Modals داخلية: `CompletionModal`, `CancelModal`, `ImageUploadSection`

### Status Tracker التصميم
```
ASSIGNED  →  ON_THE_WAY  →  STARTED  →  COMPLETED
   ●              ○              ○            ○
  مُسند        في الطريق      جارٍ          اكتمل
```
- الخطوات الماضية: دائرة ممتلئة بلون primary
- الخطوة الحالية: دائرة ممتلئة + نبضة animation
- الخطوات القادمة: دائرة فارغة بلون border

### أزرار الإجراء حسب الحالة

| الحالة الحالية | الزر | API |
|---|---|---|
| ASSIGNED | "في الطريق 🚗" | PATCH /api/biker/order/:id/status |
| ON_THE_WAY | "وصلت 📍" | PATCH /api/biker/order/:id/status |
| STARTED | "انتهيت + رفع الصور 📸" | يفتح ImageUploadSection |
| COMPLETED | لا يوجد زر | — |
| CANCELLED | لا يوجد زر | — |

## 3.3 تغيير حالة الطلب — API

```
PATCH /api/biker/order/:id/status

Body:
{
  "status": "ON_THE_WAY"   // أو STARTED أو COMPLETED
}

Statuses المتاحة للبايكر:
ASSIGNED     → ON_THE_WAY   (بدء التوجه)
ON_THE_WAY   → STARTED      (وصل وبدأ)
STARTED      → COMPLETED    (مع إرفاق الصور — proof gate)

Response (success):
{
  "success": true,
  "data": { ...updatedOrder }
}

Response (error — لا يمكن التقدم بدون صور عند COMPLETED):
{
  "success": false,
  "error": "يجب رفع صور الإثبات قبل إتمام الطلب"
}
```

## 3.4 ImageUploadSection (Modal/Sheet)

### الوصف
يظهر عند الضغط على "انتهيت" في حالة STARTED. البايكر يرفع صور ما بعد الخدمة.

### التصميم
```
┌──────── رفع صور الإثبات ──────────┐
│                                   │
│  [+ أضف صورة]  [+ أضف صورة]      │  ← Grid 2×2 أو 3×3
│  [صورة مرفوعة] [صورة مرفوعة]     │  ← preview مصغر مع × للحذف
│                                   │
│  📸 الصور الدلالية على إتمام الخدمة│
│  (مطلوبة على الأقل صورة واحدة)    │
│                                   │
│  ┌───────────────────────────┐    │
│  │  تأكيد الإتمام ✓          │    │  ← disabled حتى يرفع صورة واحدة
│  └───────────────────────────┘    │
└───────────────────────────────────┘
```

### API — رفع الصور ثم إتمام الطلب
```
PATCH /api/biker/order/:id/status

Body (multipart/form-data):
{
  "status": "COMPLETED",
  "afterPhotos": [File, File, ...]   ← array of image files
}

أو (إذا رُفعت الصور أولاً عبر Cloudinary مباشرة):
{
  "status": "COMPLETED",
  "afterPhotos": ["https://res.cloudinary.com/...", "..."]
}

Response:
{
  "success": true,
  "data": { ...completedOrder }
}
```

## 3.6 CancelModal

### الوصف
يظهر عند الضغط على "إلغاء الطلب". يتحقق من أن وقت الوصول المتوقع < 20 دقيقة.

```
┌──────────────────────────────────┐
│  ⚠️ إلغاء الطلب                  │
│                                  │
│  هل أنت متأكد من إلغاء الطلب؟    │
│  هذا الإجراء لا يمكن التراجع عنه │
│                                  │
│  [تراجع]      [نعم، إلغاء]       │
└──────────────────────────────────┘
```

### API — إلغاء الطلب
```
PUT /api/biker/order/:id

Body:
{
  "action": "cancel",
  "reason": "سبب الإلغاء"  ← اختياري
}

Response (success):
{ "success": true }

Response (error — تجاوز 20 دقيقة):
{
  "success": false,
  "error": "لا يمكن الإلغاء بعد مرور أكثر من 20 دقيقة"
}
```

## 3.7 تغيير السيارة (Change Car)

### متى يظهر؟
فقط عند حالة ASSIGNED أو ON_THE_WAY (بعد التعيين وقبل البدء).

### التصميم
```
┌──────── تغيير السيارة ────────────┐
│                                   │
│  الماركة:  [Toyota ▾]             │  ← Picker/Select
│  الموديل:  [Camry  ▾]             │  ← يعتمد على الماركة المختارة
│  اللون:    [أبيض   ▾]             │
│  رقم اللوحة: [ABC 1234]           │  ← TextInput
│                                   │
│  [ حفظ التغييرات ]                │
└───────────────────────────────────┘
```

### API
```
GET /api/biker/order/:id/change-car
Response:
{
  "brands": [
    { "_id": "...", "name": "Toyota", "models": ["Camry", "Corolla", ...] },
    { "_id": "...", "name": "Honda", "models": ["Civic", "Accord", ...] }
  ]
}

PUT /api/biker/order/:id/change-car
Body:
{
  "brand": "Toyota",
  "model": "Corolla",
  "color": "أسود",
  "plateNumber": "XYZ 5678"
}

Response:
{ "success": true, "data": { ...updatedOrder } }
```
