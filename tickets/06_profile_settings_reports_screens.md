# 06 — Profile, Settings & Reports Screens

## الشاشات المشمولة
1. `SettingsMenuScreen` — القائمة الرئيسية للإعدادات
2. `EditProfileScreen` — تعديل البيانات الشخصية
3. `ReportsListScreen` — قائمة البلاغات
4. `CreateReportScreen` — إنشاء بلاغ جديد
5. `ReportDetailScreen` — تفاصيل بلاغ


### رفع الصورة
عند الضغط على الصورة: ActionSheet بخيارين
```
"اختر من المعرض" → ImagePicker.launchImageLibrary()
"التقط صورة"     → ImagePicker.launchCamera()
```
ثم رفعها لـ Cloudinary مباشرة من الموبايل (unsigned upload):
```
POST https://api.cloudinary.com/v1_1/<cloud_name>/image/upload
Body (FormData):
{
  file: <imageUri>,
  upload_preset: "biker_profile"   ← unsigned preset مضبوط في Cloudinary
}
Response:
{ "secure_url": "https://res.cloudinary.com/..." }
```
ثم أرسل الـ URL للـ backend.

### API Endpoints

#### جلب البيانات الحالية
```
GET /api/biker/profile

Response:
{
  "success": true,
  "data": {
    "_id": "...",
    "firstName": "أحمد",
    "lastName": "محمد",
    "phoneNumber": "05xxxxxxxx",
    "email": "ahmed@example.com",
    "image": "https://res.cloudinary.com/...",
    "rating": 4.7,
    "totalReviews": 58,
    "wallet": { "balance": 1250.00 }
  }
}
```

#### حفظ التغييرات
```
PUT /api/biker/profile

Body (JSON):
{
  "firstName": "أحمد",
  "lastName": "محمد",
  "phoneNumber": "05xxxxxxxx",
  "email": "ahmed@example.com",
  "image": "https://res.cloudinary.com/..."
}

Response:
{
  "success": true,
  "data": { ...updatedUser }
}

Response (error — هاتف مكرر):
{
  "success": false,
  "error": "رقم الهاتف مستخدم بالفعل"
}
```

## ملاحظات عامة على الإعدادات

- **Dark Mode**: خزّن القيمة في AsyncStorage (`theme: 'dark' | 'light'`). استخدم Context أو Zustand لتطبيقه عبر التطبيق.
- **اللغة**: استخدم `i18next.changeLanguage('ar' | 'en')` + `I18nManager.forceRTL(true/false)` + إعادة تشغيل التطبيق (`RNRestart` مكتبة).
- **الصورة الشخصية**: اعرض placeholder (أحرف الاسم الأولى) إذا لم تكن الصورة موجودة.
